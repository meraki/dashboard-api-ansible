.. Title

cisco.meraki.networks_webhooks_payload_templates module -- Resource module for networks \_webhooks \_payload \_templates
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `cisco.meraki collection <https://galaxy.ansible.com/cisco/meraki>`_ (version 2.20.0).

    To install it, use: :code:`ansible-galaxy collection install cisco.meraki`.
    You need further requirements to be able to use this module,
    see :ref:`Requirements <ansible_collections.cisco.meraki.networks_webhooks_payload_templates_module_requirements>` for details.

    To use it in a playbook, specify: :code:`cisco.meraki.networks_webhooks_payload_templates`.

   :local:

.. Deprecated


Synopsis
--------

.. Description

- Manage operations create, update and delete of the resource networks \_webhooks \_payload \_templates.
- Create a webhook payload template for a network.
- Destroy a webhook payload template for a network. Does not work for included templates 'wpt\_00001', 'wpt\_00002', 'wpt\_00003', 'wpt\_00004', 'wpt\_00005' or 'wpt\_00006' .

- Update a webhook payload template for a network.

.. note::
    This module has a corresponding :ref:`action plugin <action_plugins>`.

Requirements
------------
The below requirements are needed on the host that executes this module.

- meraki \>= 2.4.9
- python \>= 3.5

.. Notes

Notes
-----

.. note::
   - SDK Method used are networks.Networks.create\_network\_webhooks\_payload\_template, networks.Networks.delete\_network\_webhooks\_payload\_template, networks.Networks.update\_network\_webhooks\_payload\_template,
   - Paths used are post /networks/{networkId}/webhooks/payloadTemplates, delete /networks/{networkId}/webhooks/payloadTemplates/{payloadTemplateId}, put /networks/{networkId}/webhooks/payloadTemplates/{payloadTemplateId},
   - Does not support \ :literal:`check\_mode`\ 
   - The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager
   - from Cisco Dashboard API Python(SDK)
   - The parameters starting with dnac\_ are used by the Cisco DNAC Python SDK to establish the connection
