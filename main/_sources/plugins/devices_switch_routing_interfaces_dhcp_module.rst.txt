.. Title

cisco.meraki.devices_switch_routing_interfaces_dhcp module -- Resource module for devices \_switch \_routing \_interfaces \_dhcp
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `cisco.meraki collection <https://galaxy.ansible.com/cisco/meraki>`_ (version 2.21.1).

    To install it, use: :code:`ansible-galaxy collection install cisco.meraki`.
    You need further requirements to be able to use this module,
    see :ref:`Requirements <ansible_collections.cisco.meraki.devices_switch_routing_interfaces_dhcp_module_requirements>` for details.

    To use it in a playbook, specify: :code:`cisco.meraki.devices_switch_routing_interfaces_dhcp`.

.. rst-class:: ansible-version-added

New in cisco.meraki 1.0.0

Synopsis
--------

.. Description

- Manage operation update of the resource devices \_switch \_routing \_interfaces \_dhcp.
- Update a layer 3 interface DHCP configuration for a switch.

.. note::
    This module has a corresponding :ref:`action plugin <action_plugins>`.

.. _ansible_collections.cisco.meraki.devices_switch_routing_interfaces_dhcp_module_requirements:

Requirements
------------
The below requirements are needed on the host that executes this module.

- meraki \>= 2.4.9
- python \>= 3.5

.. Notes

Notes
-----

.. note::
   - SDK Method used are switch.Switch.update\_device\_switch\_routing\_interface\_dhcp,
   - Paths used are put /devices/{serial}/switch/routing/interfaces/{interfaceId}/dhcp,
   - Does not support \ :literal:`check\_mode`\ 
   - The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager
   - from Cisco Dashboard API Python(SDK)
   - The parameters starting with dnac\_ are used by the Cisco DNAC Python SDK to establish the connection
